# Plano: Modularização GDACaixa com Vite + Vercel

## 📋 Resumo das Decisões

| Aspecto | Escolha |
|---------|--------|
| Bundler | Vite |
| Backend | Firebase Firestore (manter) |
| Hospedagem | Vercel (abandonar Firebase Hosting) |
| Domínio | gdaciaxa.page |
| Rotas | **Tabs (sem URL)** - manter estrutura atual de tabs |
| Estado | **Vanilla (APP object)** - objeto global simples |
| Branches | `main` = HTML único, `vercel` = refatorado |

## 📌 Estratégia Git

| Branch | Conteúdo |
|--------|---------|
| `main` | Mantém `index.html` original (sem alterações) |
| `vercel` | Novo código modularizado via Vite |

**Fluxo:**
1. `main` continua hospedada no Firebase Hosting (backup)
2. `vercel` faz deploy no Vercel

---

## 🎯 Objetivos

1. **Modularização**: Separar CSS, JS e HTML em arquivos organizados
2. **Versatilidade**: Estrutura escalável para adicionar módulos facilmente
3. **Organização**: Código legível e manutenível
4. **Migração**: Deploy no Vercel com domínio personalizado

## ⚠️ Considerações

### CSS (~1800 linhas)
- Extrair para arquivos por funcionalidade
- Vite processa `@import` automaticamente

### JavaScript (~3500+ linhas)
- Manter objeto global `APP` com métodos
- Cada módulo exports suas funções para `APP`
- Comunicação via callbacks/eventos simples

### Firebase
- Instalar via npm: `npm install firebase`
- Usar `.env` com `VITE_FIREBASE_*` (não commitar)
- Adicionar `.env.example`

### Build Output
- `dist/` será o diretório deployado no Vercel
- `vercel.json` redireciona tudo para `dist/index.html`

## 🔧 Configuração Inicial

### `vite.config.js`
```js
import { defineConfig } from 'vite'

export default defineConfig({
  root: '.',
  publicDir: 'public',
  build: {
    outDir: 'dist',
    sourcemap: false,
  },
  define: {
    // Substitui process.env no código
    'process.env.FIREBASE_API_KEY': JSON.stringify(process.env.VITE_FIREBASE_API_KEY),
    'process.env.FIREBASE_AUTH_DOMAIN': JSON.stringify(process.env.VITE_FIREBASE_AUTH_DOMAIN),
    'process.env.FIREBASE_PROJECT_ID': JSON.stringify(process.env.VITE_FIREBASE_PROJECT_ID),
  },
})
```

### `vercel.json`
```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

### `.env.example`
```
VITE_FIREBASE_API_KEY=your-api-key
VITE_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your-project-id
```
